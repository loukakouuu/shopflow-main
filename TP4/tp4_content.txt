1.1 Rappel du cycle TDD
RED GREEN REFACTOR
• Écrire un test qui ÉCHOUE • Écrire le MINIMUM de code • Améliorer le code
• ImportError ou • Faire passer le test, rien de proprement
AssertionError attendu plus • Sans casser les tests
• La fonction n'existe pas • Pas d'optimisation ici existants
encore • Clarté, DRY, robustesse
1.2 Contexte -Nouvelle fonctionnalité : Validation avancée
Le client ShopFlow demande des règles de validation plus strictes pour les coupons :
Spécifications de la nouvelle fonctionnalité
• valider_coupon(coupon, panier_total) → retourne True/False selon les règles ci-dessous
• Règle 1 : Le coupon doit être actif (actif=True)
• Règle 2 : La réduction doit être dans l'intervalle ]0, 100]
• Règle 3 : Le coupon ne peut être appliqué que si panier_total >= 10.0 (montant minimum)
• Règle 4 : Un coupon avec réduction=100 (gratuit) n'est valide que si panier_total >= 50.0
• Règle 5 : Lever ValueError avec message explicite pour chaque règle violée

Vulnérabilités à introduire
# Vulnérabilité 1 à introduire dans app/cache.py
# (hardcoded password — B105)

# Vulnérabilité 2 à introduire dans app/main.py
# (désactivation SSL — B501) — dans une hypothétique fonction d'appel externe
import requests
response = requests.get('https://api.externe.com', verify=False) # ← DANGEREUX
Corrections attendues
# Après correction — forme attendue
# Correction vulnérabilité 1 — utiliser os.getenv
REDIS_PASSWORD = os.getenv('REDIS_PASSWORD', '') # bonne pratique
# Correction vulnérabilité 2 — toujours verify=True
response = requests.get('https://api.externe.com', verify=True) # ok
# Ou mieux : utiliser un certificat CA personnalisé
response = requests.get('https://api.externe.com', verify='/path/to/ca.crt')
